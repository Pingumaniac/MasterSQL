How could data from a table be selected in SQL?

To select data from tables in SQL, we need to write a query in the following format:

SELECT name_of_first_attribute_to_be_selected, name_of_second_attribute_to_be_selected, ... FROM name_of_table;

You can select all attributes from the table in SQL, by writing the query in this format:

SELECT * FROM name_of_table;

For instance, if you want to select all data stored in the 'SmartPhones' table from the last example after the row for iPhone SE2 has been inserted, you need to execute the following query:

SELECT * FROM SmartPhones;

This will, in turn, return the following table:

# Model_name Manufacturer Display_size
1 iPhone 11 Mini Apple 5.4
2 iPhone 12 Mini Apple 5.4
3 iPhone 13 Apple 6.1
4 iPhone 13 Mini Apple 5.4
5 iPhone 13 Pro Apple 6.7
6 iPhone 13 Pro Max Apple 6.1
7 iPhone SE2 Apple 4.7
Note that the primary key of this table is Model_name, and hence the ordering has been automatically done in terms of Model_name.
To select whether an element exists in the database, you can use the 'EXISTS' operator.

For example, to check whether the data for iPhone 13 is stored in the table above, we can write the following query:

SELECT EXISTS(SELECT Model_name FROM SmartPhones WHERE Model_name='iPhone 13') AS Existence_of_iPhone_13;

This will return the following table.

# Existence_of_iPhone_13
1 1

Note that the 'EXISTS' operator:
  1. returns 1 when the corresponding element(s) outputted from the subquery exists in the database.
  2. returns 0 when the corresponding element(s) outputted from the subquery does not exist in the database.

If you try to check the existence of data for iPhone 14 is stored in the table above, by executing the following query:

SELECT EXISTS(SELECT Model_name FROM SmartPhones WHERE Model_name='iPhone 14') AS Existence_of_iPhone_13;

Then the following table will be returned.

# Existence_of_iPhone_14
1 0
To select distinct values of the attribute stored in the table, you need to add the term 'DISTINCT' after 'SELECT' in the query.

For instance, to select distinct display sizes in the table above, we can execute the following query:

SELECT DISTINCT Display_size FROM SmartPhones;

# Display_size
1 5.4
2 6.1
3 6.7
4 4.7
You can also order the elements in either ascending or descending order by using the "ORDER BY" keyword and "ASC" or "DESC" keyword, respectively.

For instance, to select distinct display sizes in the table above in ascending order instead, we can execute the following query:

SELECT DISTINCT Display_size FROM SmartPhones ORDER BY Display_size ASC;

# Display_size
1 4.7
2 5.4
3 6.1
4 6.7
You can also use the SQL built-in functions such as...

  1. COUNT(): to find out the number of elements in the table
  2. MIN(): to find out the minimum value of the attribute in the table
  3. MAX(): to find out the maximum value of the attribute in the table
  4. SUM(): to find out the sum of all values of the attribute in the table
  5. AVG(): to find out the average value of the attribute respectively.

For instance, we can count the number of smartphones with distinct display sizes by running the following query:

SELECT COUNT(DISTINCT Display_size) AS No_Of_SmartPhones_With_Distinct_Display_Size FROM SmartPhones;

# No_Of_SmartPhones_With_Distinct_Display_Size
1 4
Here note that it is possible to rename the attribute with an alias by employing the 'AS' command.
