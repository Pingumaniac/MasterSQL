How could tables be created in SQL?

These are the things that shall be considered when creating a table (relation):

  1. Name of the table (relation)
  2. Name and the data types of the attributes that the table shall have. Please check out the following link to see the list and descriptions for each data type for
    • MySQL
    • SQL Server
    • MS Access
  3. Constraints for each attribute. Examples include:
    • NOT NULL
    • Unique
    • Unsigned
    • Zerofill
  4. The primary key(s) of the table

Note: What is a primary key?

A primary key is an attribute or a collection of attributes that uniquely identifies each record stored in a table.

You can create a table in the following format:

CREATE TABLE (Write the table name here) (
(Name of the attribute) (type of the attribtute) (write the constraint for this attribute if necessary),
PRIMARY KEY (list the attributes used for the primary key)
);

The following is an example query for creating a table:

CREATE TABLE TutorialTable (
NameOfTutorial VARCHAR(256) NOT NULL,
Author VARCHAR(256) NOT NULL,
DateCreated TIMESTAMP NOT NULL,
PRIMARY KEY (NameOfTutorial)
);

You can also use multiple attributes as the primary key for the table:

CREATE TABLE TutorialTable (
NameOfTutorial VARCHAR(256) NOT NULL,
Type VARCHAR(256) NOT NULL,
Author VARCHAR(256) NOT NULL,
DateCreated TIMESTAMP NOT NULL,
PRIMARY KEY (NameOfTutorial, Type)
);
You can also create a table if the table does not exist yet in the database.

CREATE TABLE (Write the table name here) IF NOT EXISTS (
(Name of the attribute) (type of the attribtute) (write the constraint for this attribute if necessary),
PRIMARY KEY (list the attributes used for the primary key)
);

The following is an example query for creating a table:

CREATE TABLE TutorialTable IF NOT EXISTS (
NameOfTutorial VARCHAR(256) NOT NULL,
Author VARCHAR(256) NOT NULL,
DateCreated TIMESTAMP NOT NULL,
PRIMARY KEY (NameOfTutorial)
);
