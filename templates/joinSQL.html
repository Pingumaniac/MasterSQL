How could two or more tables be joined to retreive data in SQL?

There are four main types of JOIN in SQL:
  1. INNER JOIN
  2. LEFT JOIN (or LEFT OUTER JOIN)
  3. RIGHT JOIN (or RIGHT OUTER JOIN)
  4. FULL JOIN (or FULL OUTER JOIN)

Note that
  1. INNER JOIN returns the attributes from both tables that satisfy the matching condition.
  2. LEFT JOIN returns all the attributes from the LEFT table and all the attributes for the RIGHT table, which satisfies the matching condition given. If the matching condition from the RIGHT table is not satisfied, then null will be returned for the value(s) of the attribute(s) from the RIGHT table.
  3. RIGHT JOIN returns all the attributes from the RIGHT table and all the attributes for the LEFT table, which satisfies the matching condition given. If the matching condition from the LEFT table is not satisfied, then null will be returned for the value(s) of the attribute(s) from the LEFT table.
  4. FULL JOIN returns all the attributes from both sides. If the matching condition from any table is not satisfied, then null will be returned for the corresponding attribute(s).
  5. MySQL does not support FULL JOIN. The results from LEFT JOIN and RIGHT JOIN shall be united using the 'UNION' clause to join the table fully.

To join two or more tables in SQL for retrieving data from them, you need to write a query in the following format:

SELECT (write the table name and their attributes that you want to select as Table_name.column_name) FROM Original_Table (write the join clause that you want to use here) Table_to_be_joined ON (Write the condition here)

For instance, assume that there are two tables, DBMS and HCI, containing the student IDs, names, and final grades of the students who took Database Management Systems (DBMS) and Human-Computer Interaction (HCI), respectively.

Table name: DBMS
# StudentID StudentName Grade
1 1001 Aaron A+
2 1002 Bash A
3 1003 Claire A-
4 1004 Darwin B+
5 1005 Euler B

Table name: HCI
# StudentID StudentName Grade
1 1001 Aaron A
2 1002 Bash A-
3 1003 Claire A+
4 1006 Fortran C+
5 1007 Haskell F
To select the student IDs and the names of students, the grades for the students who took both courses, we can execute the following query:

SELECT DBMS.StudentID AS StudentID, DBMS.StudentName AS StudentName, DBMS.Grade, HCI.Grade FROM DBMS INNER JOIN HCI ON DBMS.StudentID = HCI.StudentID;

The following table will be outputted.

# StudentID Name DBMS.Grade HCI.Grade
1 1001 Aaron A+ A
2 1002 Bash A A-
3 1003 Claire A- A+

If we use LEFT JOIN instead of INNER JOIN in the query above, the following table will be outputted instead:

# StudentID Name DBMS_Grade HCI_Grade
1 1001 Aaron A+ A
2 1002 Bash A A-
3 1003 Claire A- A+
4 1004 Darwin B+ null
5 1005 Euler B null

If we execute the following query with RIGHT JOIN:

SELECT HCI.StudentID AS StudentID, HCI.StudentName AS StudentName, DBMS.Grade AS DBMS_Grade, HCI.Grade AS HCI_Grade FROM DBMS RIGHT JOIN HCI ON DBMS.StudentID = HCI.StudentID;

Then, the following table will be outputted.

# StudentID StudentName DBMS_Grade HCI_Grade
1 1001 Aaron A+ A
2 1002 Bash A A-
3 1003 Claire A- A+
4 1006 Fortran null C+
5 1007 Haskell null F

We can also unite the results from LEFT JOIN and the results from RIGHT JOIN to create the same effect as FULL JOIN in MySQL as follows:

SELECT DBMS.StudentID AS StudentID, DBMS.StudentName AS StudentName, DBMS.Grade, HCI.Grade FROM DBMS LEFT JOIN HCI ON DBMS.StudentID = HCI.StudentID
UNION
SELECT HCI.StudentID AS StudentID, HCI.StudentName AS StudentName, DBMS.Grade AS DBMS_Grade, HCI.Grade AS HCI_Grade FROM DBMS RIGHT JOIN HCI ON DBMS.StudentID = HCI.StudentID;

Then, the following table will be outputted instead:

# StudentID StudentName DBMS_Grade HCI_Grade
1 1001 Aaron A+ A
2 1002 Bash A A-
3 1003 Claire A- A+
4 1004 Darwin B+ null
5 1005 Euler B null
6 1006 Fortran null C+
7 1007 Haskell null F
