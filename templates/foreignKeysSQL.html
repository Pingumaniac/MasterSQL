What is a foreign key constraint in SQL?

A foreign key is a set of attributes in one (child) table that refers to another attribute from another (parent) table.

Note that the child table's foreign key shall correspond to the parent table's primary key.

A case in point, the foreign key constraint can be defined as follows:

CREATE TABLE IF NOT EXISTS Consumer (
consumerID VARCHAR(256) NOT NULL,
consumerFullName VARCHAR(256) NOT NULL,
PRIMARY KEY (consumerID)
);

CREATE TABLE IF NOT EXISTS Product (
productID VARCHAR(256) NOT NULL,
productName VARCHAR(256) NOT NULL,
PRIMARY KEY (productID)
);

CREATE TABLE IF NOT EXISTS Purchased (
consumerID VARCHAR(256) NOT NULL,
productID VARCHAR(256) NOT NULL,
PRIMARY KEY (consumerID, productID),
FOREIGN KEY (consumerID) REFERENCES Consumer(consumerID),
FOREIGN KEY (productID) REFERENCES Product(productID)
);

Notice that the 'Purchased' table is a child table that has'Consumer' and 'Product' tables as its parent tables.
